# 卡背 → 徽章轉換:方向紀錄

日期:2026-07-14(進行中,明天接力)
素材:`what-do-you-think-is-the-most-impressive-cardback-v0-wgyrlt122fcd1.jpg`(1372×991,爐石卡背收藏頁截圖,4×2 共八張卡背)
原型:`prototype-badges.html`(repo 根目錄,瀏覽器直接開;headless 驗證需 `--allow-file-access-from-files`)

## 目標

把卡背的美術轉換成徽章(或其他非卡片形狀)呈現。硬需求:

1. **一眼看出同款** — 必須直接取用原圖像素,不是「風格類似」。
2. **不是卡片形狀** — 要脫離長方形卡背的載體。
3. **不生硬** — 不能像「從卡片上剪一塊下來」。

## ✅ 採用方向:輪廓萃取(shape-from-art)

**形狀由畫面元素決定,不是外加的幾何形狀。**

每張卡背的中央徽記本來就是一個現成的「徽章物件」(CARD 08 的尖刺圓盤、
熔岩卡的火焰漩渦、六角卡的金屬徽、眼球卡的眼睛)。做法:

- 生成貼合該物件輪廓的遮罩(例:盤面圓 + 逐根對位的尖刺三角),SVG `<mask>` 套在原圖上
- 遮罩整組過 `feGaussianBlur`(stdDeviation ≈ 2)輕微羽化,消除剪貼感
- 成品剪影不規則、天生像徽章,且像素 100% 來自原畫
- 八張卡各自會得到形狀完全不同的徽章,不是八個同形狀裁切

輔助手法(可疊加,原型編輯器都已實作):

- **裝飾件軟邊萃取**:原圖任意位置的細節(頭骨、寶石…)用放射漸層 mask
  淡出邊緣摘出來,疊在主體上當裝飾(帶 drop-shadow 做層次)
- **幾何形狀裁切**:仍保留(圓/封蠟章/八芒星/盾/六角/菱形/火焰),需要規整形狀時用

## ❌ 不要的方向(已驗證、已否決)

1. **風格重繪** — 萃取設計元素轉成文字描述,再手寫 SVG 重畫。
   結果只是「看起來風格類似」,認不出是同款。**使用者明確否決。**
2. **幾何形狀硬裁(截圖思維)** — 圓形/盾形等固定形狀直接從原圖裁下來。
   同款辨識沒問題,但像「從卡片拔一塊」,生硬。軟邊淡出只是緩解,
   思維仍是「圈一塊圖」,同樣否決。
3. (合成徽章 = 硬裁主體 + 裝飾件,仍屬截圖思維,效果不夠好 → 演化成輪廓萃取)

## 原型編輯器現況(prototype-badges.html)

使用者已批准保留的功能:

- 圖層系統:主徽記 + N 個裝飾件,可新增/刪除/選取(虛線框標示)
- 每個裝飾件兩種編輯模式:**取材**(動原圖裁切 cx/cy/r)、**擺放**(動徽章上位置 px/py/size)
- 拖曳移動、滾輪縮放、方向鍵微調(Shift=5px)、滑桿
- 形狀選單(僅主徽記)、即時縮放預覽(120/72/40/24px)
- 參數即時顯示 + 複製,格式:`{cx, cy, r, shape}` / `{cx, cy, r, px, py, size}`
- 頁面最下方:輪廓萃取示範區(spikyPlateSVG)

## 技術備忘

- **`clipPath` 內不能放 `<g>`**(會被整個忽略→裁切區變空),transform 要直接放在形狀元素上
- headless Chrome 驗證這頁要加 `--allow-file-access-from-files`(SVG `<image>` 載本地圖)
- 羽化 = mask 內容套 `feGaussianBlur`;軟邊圓 = mask 填 userSpaceOnUse 的 radialGradient
- 尖刺對位目前約九成準,少數刺帶到背景像素,靠暗背景+羽化融掉;要完美就逐根微調參數
